# PulseERP v3.9.0 — Stabilisation priorité 1

## Corrections

- Pages 404 et 500 propres
- Écran de chargement global
- Erreur dédiée au back-office développeur
- Validation Zod des tâches
- Attribution réelle des tâches aux utilisateurs
- Validation Zod des opérations comptables
- Retours utilisateur après création
- Correction du sélecteur Source CRM
- Endpoint `/api/health/runtime`
- Scripts de contrôle TypeScript et Prisma
- Aucun changement Stripe

## Installation

```bash
rm -rf .next node_modules
npm install
npx prisma migrate deploy
npm run stability:check
npm run verify
```

## Déploiement

```bash
git add .
git commit -m "release: stabilisation PulseERP v3.9"
git push origin main
npx vercel --prod --force
```

## Contrôles de production

Ouvrir :

```text
/api/health/config
/api/health/runtime
```

Les deux endpoints doivent retourner `"ok": true`.

## Parcours critiques

- `/register` : création d’un client
- `/login` : connexion client
- `/developer/login` : connexion développeur
- `/admin` : gestion des entreprises
- `/contacts` : création et modification
- `/acquisition` : dépenses et coût par lead
- `/tasks` : création avec attribution
- `/calendar` : création et modification
- `/billing` : devis et factures
- `/transactions` : opération manuelle
- `/dashboard` : revenu manuel
- `/settings` : entreprise et compte
