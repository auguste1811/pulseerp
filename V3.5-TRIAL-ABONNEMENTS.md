# PulseERP v3.5.0 — Essai 3 jours et abonnements

## Parcours

1. L’utilisateur crée son entreprise.
2. Un essai gratuit de trois jours démarre automatiquement, sans carte.
3. Une bannière affiche le temps restant.
4. À l’expiration, toutes les pages et actions utilisant `currentContext()`
   redirigent vers `/subscribe`.
5. L’utilisateur choisit Starter, Pro ou Business.
6. Stripe Checkout collecte le paiement.
7. Le webhook Stripe active l’entreprise.
8. Le portail Stripe permet de gérer la carte, les factures et la résiliation.

Le compte de démonstration reste actif et n’expire pas.

## Produits Stripe

Dans Stripe, crée trois produits récurrents mensuels :

- Starter : 19 EUR HT/mois
- Pro : 39 EUR HT/mois
- Business : 79 EUR HT/mois

Copie les identifiants `price_...` dans les variables correspondantes.

## Webhook Stripe Billing

URL :

```text
https://VOTRE-DOMAINE/api/webhooks/stripe-billing
```

Événements :

```text
checkout.session.completed
customer.subscription.created
customer.subscription.updated
customer.subscription.deleted
invoice.paid
invoice.payment_failed
```

Utilise un endpoint séparé du webhook Stripe Connect, donc un secret distinct :

```env
STRIPE_BILLING_WEBHOOK_SECRET=whsec_...
```

## Installation

```bash
npm install
npx prisma migrate deploy
npx prisma generate
npm run build
```

## Test

En mode Test Stripe :

1. crée un compte PulseERP ;
2. vérifie la bannière d’essai ;
3. modifie temporairement `trial_ends_at` dans Prisma Studio pour simuler
   l’expiration ;
4. recharge une page interne ;
5. choisis une offre ;
6. utilise la carte Stripe de test `4242 4242 4242 4242` ;
7. vérifie que le webhook place l’abonnement en `ACTIVE`.
