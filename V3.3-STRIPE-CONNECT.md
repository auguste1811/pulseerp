# PulseERP v3.3.0 — Stripe Connect

## Expérience client

Le client ouvre **App Center → Stripe → Connecter Stripe**. Il est redirigé
vers le formulaire Stripe hébergé, se connecte ou crée son compte puis revient
dans PulseERP. Aucune clé API client n’est demandée.

## Variables serveur

```env
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_CONNECT_COUNTRY=FR
APP_URL=http://localhost:3000
```

En production, `APP_URL` doit être le domaine Vercel public.

## Installation

```bash
npm install
npx prisma generate
npm run build
```

Aucune migration de base n’est requise : le connecteur utilise la table
`integration_connections` existante.

## Configuration Stripe

1. Activer Connect dans le Dashboard Stripe de la plateforme.
2. Compléter la marque Connect : nom, couleur, icône et informations légales.
3. Ajouter un endpoint webhook :

```text
https://VOTRE-DOMAINE.vercel.app/api/webhooks/stripe
```

4. Écouter les événements du compte de plateforme **et des comptes connectés** :
   - `account.updated`
   - `checkout.session.completed`
   - `checkout.session.async_payment_succeeded`
5. Copier le secret `whsec_...` dans `STRIPE_WEBHOOK_SECRET`.

## Paiement des factures

Sur une facture non payée, le bouton **Créer le paiement Stripe** ouvre Stripe
Checkout sur le compte connecté de l’entreprise. Le webhook marque ensuite la
facture et le mouvement comptable comme payés.

## Test

Utiliser les clés Stripe en mode test et une carte de test Stripe. Ne passer en
mode production qu’après validation complète de l’onboarding et des webhooks.
