# PulseERP v3.4.0 — OCR Pro

## Fonctionnement

1. Importer une facture PDF, JPG, PNG ou WebP.
2. Cliquer sur `Analyser OCR`.
3. PulseERP lit le document avec l’API OpenAI.
4. Une page de contrôle affiche le document et les valeurs détectées.
5. L’utilisateur corrige les éventuelles erreurs.
6. La validation met à jour la facture, la dépense et les écritures.

L’analyse ne modifie plus directement la comptabilité. Une validation humaine
est obligatoire.

## Installation

```bash
npm install
npx prisma migrate deploy
npx prisma generate
npm run build
```

## Variables

```env
OPENAI_API_KEY=...
OPENAI_MODEL=gpt-4.1-mini
OPENAI_OCR_MODEL=gpt-4.1-mini
```

`OPENAI_OCR_MODEL` est facultative. Si elle est absente, PulseERP utilise
`OPENAI_MODEL`.

## États OCR

- `NOT_ANALYZED`
- `PROCESSING`
- `READY`
- `VALIDATED`
- `ERROR`

## Données détectées

- fournisseur
- adresse
- numéro de TVA
- SIRET
- numéro de facture
- dates
- devise
- HT
- TVA
- TTC
- taux de TVA
- catégorie
- référence de paiement
- IBAN
- score de confiance
- avertissements
